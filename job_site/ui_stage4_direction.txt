4. RESPONSIVE RULES (pinned)
----------------------------------------------------------
Desktop (>= 769 px)
  - Two-column panel grid (1fr 1fr).
  - Full status bar with Core | State | Session | Calendar + toggle
    + last-event-marker.
  - Voice footer: badge + Talk + (Stop & Send | Cancel) + message.
  - Calendar drawer: 380 px wide, docked right.

Tablet / narrow desktop (<= 768 px)
  - Panels stack vertically (two rows).
  - Status bar compresses: dividers hidden, last-event-marker
    hidden (state pill + calendar status still visible).
  - Voice footer: reduced horizontal padding.

Phone (<= 640 px)
  - Calendar drawer goes full-screen (width: 100vw, no left border).

Small phone (<= 420 px)
  - Voice button text labels collapse to icon-only via
    `.voice-btn-label { display: none }`. Icons and aria-labels
    keep the buttons identifiable and screen-reader-correct.

Horizontal-scroll fallback: the status bar uses `overflow-x: auto`
so any extreme label drift degrades gracefully instead of
truncating.

6. STATE-COLOUR CONTRACT (cross-surface)
----------------------------------------------------------
The same colour means the same thing everywhere.

  live / ready / ok      → green (--ok)
  loading / not-ready /
   unconfigured / warn   → amber (--warn)
  error / fail /
   unsupported / denied  → red (--bad)
  idle / empty / unknown → grey (--text-dim)

This applies to:
