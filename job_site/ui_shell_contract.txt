CONTRACT 1 — CHAT INPUT
────────────────────────────────────────────────────────────────────────────────
Required DOM element:
  <input type="text" id="chat-input">
  Placement: fixed chat bar above voice footer
  aria-label="Type a message"
  placeholder="Type a message…"
  onkeydown: fires submit() on Enter (no Shift)

Required DOM element (send button):
  <button id="send-btn">Send</button>
  aria-label="Send message"
  Disabled state: during in-flight send; re-enabled in finally block

Behavior contract:
  1. Operator types text into #chat-input
  2. Operator clicks #send-btn or presses Enter (no Shift)
  3. submit() in index.html inline module calls sendMessage() from session-view.js
       payload: { text: <trimmed input value>, source: "browser" }
  4. POST /ui/send  (SEND_ENDPOINT = '/ui/send' — session-view.js:29)
       Content-Type: application/json
  5. On { ok: true }: input cleared, focus restored, exchange visible in
       #transcript-log on next poll (≤5 s)
  6. On { ok: false }: visible error shown to operator; input preserved
  7. No placeholder-only replies acceptable — assistant must return a real reply

Route:       POST /ui/send
Route owner: worker-wb/src/routes/ui-send.ts

DOM presence (confirmed in worker-wb/public/index.html):  PRESENT

CONTRACT 2 — TALK BUTTON (voice start)
────────────────────────────────────────────────────────────────────────────────
Required DOM element:
  <button class="voice-btn voice-btn-start" id="voice-btn-start">
  aria-label="Start recording"
  Visible text: "Talk" (collapses to mic glyph at ≤420px)

Behavior contract:
  1. On click: calls startCapture() from voice-capture.js
  2. startCapture() calls navigator.mediaDevices.getUserMedia({ audio: true })
  3. Browser must show native mic permission prompt on first click
  4. State transitions:
       idle → requesting → recording   (permission granted)
       idle → requesting → denied      (permission denied)
       idle → requesting → denied      (no device found)
       idle → requesting → unsupported (MediaDevices API absent)
  5. #voice-status-badge reflects current state (aria-live="polite")

DOM presence (confirmed in worker-wb/public/index.html):  PRESENT

CONTRACT 3 — STOP/SEND AND RECORD CONTROLS
────────────────────────────────────────────────────────────────────────────────
Required DOM elements:
  #voice-btn-stop    "Stop & Send"  aria-label="Stop and send recording"
  #voice-btn-cancel  "Cancel"       aria-label="Cancel recording"

Behavior contract:
  Stop & Send:
    Wired to stopAndSend() in voice-capture.js
    Transitions: recording → processing → sent → idle (auto-reset 3 s)
    POST /voice/capture  (VOICE_ENDPOINT = '/voice/capture' — voice-capture.js:23)
      Body: multipart/form-data, field "audio", filename "recording.webm"
      MIME: audio/webm;codecs=opus preferred; webm/ogg/mp4 fallback
    Success: { ok: true, session_id?: string }
    Failure: { ok: false, error: { code: string, message: string } }
  Cancel:
    Wired to cancelCapture() in voice-capture.js
    Transitions: recording → idle (no upload)

  Visibility:
    Stop & Send: hidden when not recording; shown only during recording
    Cancel:      hidden at idle; shown during recording and terminal states

Route:       POST /voice/capture
Route owner: worker-wb/src/routes/voice-capture.ts

DOM presence (confirmed in worker-wb/public/index.html):  PRESENT

CONTRACT 4 — CALENDAR ENTRY (browser surface)
────────────────────────────────────────────────────────────────────────────────
Status bar entry:
  #status-calendar       — live pill: "demo ready" / "demo not ready" / "—"
                           data-state="live" (green) | data-state="error" (red)
  #calendar-panel-toggle — "⚙ Calendar" button; toggles drawer on click

Calendar drawer (#calendar-panel):
  Auto-refreshes /calendar/status on open

Drawer DOM hooks:
  #calendar-panel-provider     — provider name string
  #calendar-panel-state        — state pill (six closed values below)
  #calendar-panel-reason       — hidden unless reason present
  #calendar-panel-detail       — hidden unless detail present
  #calendar-panel-guidance     — shown for unconfigured/unsupported/not_ready/error
  #calendar-panel-action-type  — static label "calendar.list_today"
  #calendar-panel-run          — "Run calendar.list_today" button
                                  disabled only when state === "unknown"
                                  enabled for all other states including unconfigured
  #calendar-panel-result       — result container, hidden until invocation
  #calendar-panel-result-state — "ok" or "fail" pill
  #calendar-panel-result-reply — reply_text from backend
  #calendar-panel-result-meta  — "handled_at <ts> · N events" or error code
  #calendar-panel-error        — shown only on ERROR state

Action bar (outside drawer):
  #calendar-action-bar    — hidden until invocation present
  #calendar-action-result — last action result text

Six closed UI states:
  unknown       — not fetched; pill "—"
  ready         — provider.ready === true; green "configured · ready"
  not_ready     — provider.ready === false, reason ≠ provider_not_configured
                  yellow "configured · not ready"
  unconfigured  — reason === provider_not_configured or no provider
                  muted "unconfigured"
  unsupported   — reason === provider_unsupported; red "unsupported provider"
  error         — route unreachable / non-200 / malformed; red "unreachable"

Routes:
  GET /calendar/status       — read provider state, no side effects
  GET /calendar/status?run=1 — dispatch calendar.list_today, return result
  Route constant: CALENDAR_ENDPOINT = '/calendar/status' (session-view.js:28)

Route owner: worker-wb/src/routes/calendar-status.ts

DOM presence (confirmed in worker-wb/public/index.html):  PRESENT
