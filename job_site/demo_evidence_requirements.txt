4. FORMAT RULES
====================================================================
- JSON evidence MUST be valid JSON (no HTML wrapper, no curl meta).
  If a response is non-JSON, record it as { "raw": "<text>" }.
- PNG evidence MUST be full-pane screenshots, not cropped snippets,
  so the status bar and pane headers stay visible.
- Every evidence file MUST include a timestamp within the screenshot
  or as a sibling JSON file with { "captured_at": "<ISO8601>" }.
- No evidence file may contain live secrets (BOT_TOKEN, HOOK_SECRET,
  DEEPGRAM_API_KEY). Redact to "<REDACTED>" before committing.
- Redactions must be visible in the screenshot (opaque bar), not
  cropped away, so the reviewer can confirm a field existed.
