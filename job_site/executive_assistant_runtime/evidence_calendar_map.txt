--- 2d. CANCEL PATH PREVENTS WRITE ---

  Source: calendar_actions.py:25-32 (docstring) + AssistantCore._confirmation_path
  Mechanism:
    When user replies "no", AssistantCore._confirmation_path(confirmed=False)
    clears pending_action from the session and returns a cancellation message.
    handle_create() is NEVER called again — so provider.create_event() is
    never invoked.  The stale _pending_creates entry is harmlessly overwritten
    on the next create request from that session.
