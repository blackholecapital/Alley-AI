  Client (_sendAudio, voice-capture.js:181-200):
    - fetch('/voice/capture', { method: 'POST', body: FormData })
    - FormData field "audio", filename "recording.webm"
    - HTTP non-2xx → { ok:false, error:{ code:'http_<status>', ... } }
    - Non-{ok:true} body → { ok:false, error:{ code:'bad_response' } }
    - fetch rejection → { ok:false, error:{ code:'network' } }
    All three error codes match contract §ERROR HANDLING RULES.

  Server (handleVoiceCapture, voice.ts:394-508):
    - POST only; other methods return 405 errorResponse envelope.
    - Rejects when audio field missing / not a File → 400.
    - Rejects unsupported MIME → 415.
    - Rejects 0-byte audio → 400.
    - Rejects > DEFAULT_MAX_VOICE_BYTES → 413.
    - Success → 200 { ok:true, session_id:<string|null>, correlation_id }
    - Appends a single recordUiInbound entry (text = transcript or
      "[voice: mime, size bytes]" stub).

  Every response is the shared JSON envelope (jsonResponse /
  errorResponse); contract expectations hold.

  RESULT: endpoint contract satisfied on both sides.
