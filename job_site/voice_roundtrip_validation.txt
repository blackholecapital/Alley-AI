5. MAPPING TO PIPELINE-STATE SOURCE
----------------------------------------------------------------------
/worker-wb/src/routes/session-latest.ts (buildPipelineView):
  items.length === 0           → current_state = "idle"
  newest.direction === inbound → current_state = "received"
  newest.kind     === failure  → current_state = "failed"
  else (outbound, kind=text)   → current_state = "replied"

Post-patch handleVoiceCapture (voice.ts, e8f466d) guarantees that
every accepted POST writes the INBOUND first and a completion row
(text or failure) second. Because getLatest returns items newest-first,
the completion row is items[0] and therefore drives current_state.
This closes the gap the prior pass flagged.
