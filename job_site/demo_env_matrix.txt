4. VALIDATION BEHAVIOR
----------------------------------------------------------
src/lib/env.ts — validateTelegramEnv() / validateTranscriptionEnv():
   Called per-request by /telegram/webhook and /voice/capture.
   Returns EnvValidation (ok:bool, errors:[], warnings:[]). Callers
   return 500 config_error on any errors; warnings are logged but
   allow the request to proceed.

   validateTelegramEnv:
     TELEGRAM_BOT_TOKEN missing       → error → 500 on /telegram/webhook
                                                  and /voice/capture reply path
     TELEGRAM_WEBHOOK_SECRET missing  → error → 500 on /telegram/webhook only

   validateTranscriptionEnv:
     provider=deepgram + no API key   → error → 500 on voice path only
     provider=noop or unset           → warn  → voice path degrades safely
     provider=unknown string          → error → 500 on voice path only

src/integrations/calendar/provider.ts — getCalendarProvider(env):
   Called per-request by /calendar/status. Does NOT throw. Returns:
     DemoCalendarProvider (ready=true) when CALENDAR_PROVIDER="demo"
     UnconfiguredCalendarProvider (ready=false, reason=provider_not_configured)
       when CALENDAR_PROVIDER is unset, empty, "none", or "unconfigured"
     not-ready object (reason=provider_unsupported)
       when CALENDAR_PROVIDER is any other string (e.g. "google")

   /calendar/status always returns HTTP 200. Handler returns ok=false with
   human-readable reply_text when provider not ready. Isolate never crashes.

   Stage 4 addition: every /calendar/status response includes a `ui` block
   derived by buildCalendarUiView(providerStatus, lastResult):
     ui.provider.state  ∈ ready | not_configured | unsupported | not_ready
     ui.trigger.enabled = providerStatus.ready
     ui.last_result     mirrors the most recent invocation result (persists
                        across passive polls)

6. BROWSER VOICE PERMISSIONS (client-side, no env key)
----------------------------------------------------------
The browser voice capture path requires the user's browser to grant
microphone access. This is independent of Worker env vars. States:

  unsupported   — navigator.mediaDevices not available (non-HTTPS or
                  very old browser). Voice capture button hidden.
                  No Worker call made.

  denied        — User denied mic permission (or browser policy blocks).
                  #voice-status-badge shows data-state="error".
                  No Worker call made.

  idle          — Mic available; capture not started.
                  #voice-status-badge data-state="idle" or hidden.

  recording     — Browser capturing audio (MediaRecorder active).
                  #voice-status-badge data-state="recording" with
                  rec-pulse animation.
                  NOT visible in /session/latest (client-side only).

  processing    — Audio sent to POST /voice/capture; awaiting reply.
                  #voice-status-badge data-state="processing".
                  NOT visible in /session/latest until Worker records
                  inbound item (pipeline recording addition in Stage 5+).

  sent          — Reply received from Worker. Badge resets to idle.

  error         — POST /voice/capture returned non-2xx or network error.
                  #voice-status-badge data-state="error".

Operator note: browser voice requires HTTPS. wrangler dev uses HTTP
localhost; voice capture will show "unsupported" on plain HTTP unless
the browser allows localhost as a secure origin.

8. OBSERVABILITY
----------------------------------------------------------
wrangler.jsonc has "observability.enabled": true. All logs via
src/lib/logging.ts are captured. Each JSON line carries at minimum:
  ts, level, event, correlation_id, env, version, method, path

Correlation IDs flow: cf-ray → x-correlation-id → x-request-id →
crypto.randomUUID(). Same ID appears in response body and
x-correlation-id header.

Key log tags (Stage 4):
  session.latest.served       — pipeline_state included in log fields
  calendar.status.poll        — triggered_by, ui_provider_state
  calendar.status.run         — invocation.ok, ui_last_result_state
  voice.capture.received      — audio_bytes, transcription_provider
  voice.capture.transcribed   — transcript_length
  voice.capture.reply_sent    — reply_text length
